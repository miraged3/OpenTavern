# 角色卡与聊天指南

OpenTavern 兼容 SillyTavern 生态中的常见角色卡格式，并围绕角色组织对话。

## 支持的角色卡格式

OpenTavern 当前支持：

- PNG 角色卡：从 PNG 元数据中读取 `chara` 或 `ccv3` 字段。
- JSON V1：顶层为角色字段的传统 JSON 角色卡。
- JSON V2 / V3：`spec` 为 `chara_card_v2` 或 `chara_card_v3`，并包含 `data` 对象。

导入 PNG 角色卡时，应用会保留 PNG 图片作为角色头像。导入 JSON 角色卡时，如果 JSON 中包含头像数据或相关字段，应用会尽可能保留可识别信息。

## 导入角色

1. 打开“角色”页。
2. 选择导入入口。
3. 选择 PNG 或 JSON 角色卡文件。
4. 检查导入后的角色信息。
5. 保存角色。

如果导入失败，常见原因包括：

- 文件不是有效 PNG 或 JSON。
- PNG 中没有 `chara` / `ccv3` 元数据。
- JSON 顶层不是对象。
- 角色卡字段不完整或来自非兼容格式。

## 角色信息

角色卡通常包含名称、描述、性格、开场白、示例对话等信息。OpenTavern 会把这些字段组织成模型提示词，用于让模型按照角色设定回复。

角色设定越清晰，模型越容易保持一致：

- 名称和身份要明确。
- 性格描述避免互相矛盾。
- 开场白可以体现语气和场景。
- 示例对话可以帮助模型学习说话方式。

## 开始聊天

1. 确认已经配置并启用至少一个模型端点。
2. 打开角色详情或聊天入口。
3. 选择角色开始新对话。
4. 输入消息并发送。

聊天时应用会把角色设定、当前对话上下文和用户消息发送给你配置的模型服务。远程模型服务可能会根据其自身政策记录或处理请求，请按服务商说明管理隐私风险。

## 消息操作

OpenTavern 的聊天界面支持常见消息操作，包括：

- 流式输出：模型回复会逐步显示。
- 编辑消息：修改已有消息内容。
- 重新生成：让模型基于当前上下文重新回复。
- 上下文控制：管理发送给模型的历史内容。
- 分支对话：在不同回复之间保留分支上下文。

具体可用操作会随页面状态变化。例如模型正在回复时，部分编辑或重新生成操作可能暂时不可用。

## 用户人格

用户人格用于描述“你”在对话中的身份、语气或背景。配置后，模型可以基于这些信息调整对你的称呼和互动方式。

建议：

- 保持简短清晰。
- 只写希望模型长期记住的信息。
- 不要把 API Key、密码或其他敏感信息写入人格。

## 隐私与数据

OpenTavern 默认本地保存角色、对话和设置。需要注意：

- 使用云端模型时，对话内容会发送到对应服务商。
- 使用本地模型时，内容只发送到你配置的本地或自托管服务。
- 卸载应用或清理应用数据可能删除本地保存内容。

在导入来自互联网的角色卡前，建议检查角色设定中是否包含不希望发送给模型服务的敏感内容。
